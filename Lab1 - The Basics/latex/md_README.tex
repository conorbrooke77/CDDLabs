\Hypertarget{md_README_autotoc_md0}%
 This lab demonstrates the concepts of mutual exclusion and thread synchronization using semaphores. We explore two main scenarios\+:


\begin{DoxyEnumerate}
\item Synchronizing two threads so that one thread prints its messages before the other.
\item Ensuring that only one of many threads can update a shared variable at a given time, demonstrating mutual exclusion.
\end{DoxyEnumerate}\hypertarget{md_README_autotoc_md1}{}\doxysection{\texorpdfstring{Description}{Description}}\label{md_README_autotoc_md1}
\hypertarget{md_README_autotoc_md2}{}\doxysubsection{\texorpdfstring{1. Semaphore-\/based Synchronization}{1. Semaphore-based Synchronization}}\label{md_README_autotoc_md2}
Two threads are launched\+: one displaying a message after a delay and another displaying its message upon receiving a signal from the first thread. This ensures that the messages are always printed in the correct order, regardless of how the threads are scheduled.\hypertarget{md_README_autotoc_md3}{}\doxysubsection{\texorpdfstring{2. Mutual Exclusion with a Shared Variable}{2. Mutual Exclusion with a Shared Variable}}\label{md_README_autotoc_md3}
A large number of threads ({\ttfamily num\+\_\+threads}) are launched concurrently. Each thread increments a shared variable multiple times. A semaphore ensures mutual exclusion so that only one thread updates the shared variable at any given moment. After all threads have finished, the shared variable\textquotesingle{}s final value shows the succussful implementation of mutual exclusion.\hypertarget{md_README_autotoc_md4}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_README_autotoc_md4}
